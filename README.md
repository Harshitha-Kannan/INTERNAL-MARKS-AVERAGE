# 📘 Internal Marks Average Management System

A full-stack web application designed to digitize and streamline the management of internal assessment records in educational institutions.

This system provides secure role-based access for **Students, Professors, and Class In-Charges**, ensuring structured academic record management, transparency, and improved administrative efficiency.

---

## 🚀 Project Overview

The **Internal Marks Average Management System** eliminates manual record maintenance by providing a centralized digital platform for managing internal assessment data.

Users are authenticated through a login system and redirected to **role-specific dashboards** where they can perform designated tasks such as:

- Viewing records  
- Entering marks  
- Calculating averages  

---

## 🎯 Objectives

- Implement role-based authentication  
- Manage internal assessment records digitally  
- Reduce manual calculation errors  
- Store and retrieve academic data efficiently  
- Demonstrate practical implementation of DBMS concepts  

---

## 👥 User Roles & Features

### 👨‍🎓 Student Module

- Secure login access  
- View internal marks and subject details  
- Monitor academic performance  

---

### 👨‍🏫 Professor Module

- Enter roll number, student name, and subject  
- Record **Part A (1–5)** marks  
- Record **Part B (6a–10b)** marks  
- Calculate and submit total marks  
- Store records in MongoDB  

---

### 👩‍💼 Class In-Charge Module

- Enter subject marks  
- Automatically calculate average  
- Display computed average  
- Store average record in database
